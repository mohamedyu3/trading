# SADDAM Zone EA V1.11 - Trading Flow & Logic

This document provides a technical breakdown of the core logic and operational flow of version 1.11, which serves as the foundation for all subsequent versions.

## 1. Operating Modes
The EA operates in two distinct modes based on the risk level (Total Lots):

### A. Normal Mode (Grid-Martingale)
*   **Trigger:** Total Buy or Sell lots are below the `MaxLots` threshold.
*   **Goal:** Profit generation through cost-averaging.
*   **Flow:**
    1.  **Initial Entry:** Opens a single trade (`FirstOrder`) if no orders exist.
    2.  **Grid Placement:** Places a pending "Stop" order (`BUYSTOP` or `SELLSTOP`) in the opposite direction of the last trade at a distance of `Step` pips.
    3.  **Multiplier:** Each new pending order uses the `Multiplier` to increase volume, helping to recover losses when the price reverses.
    4.  **TP Synchronization:** Every time a new trade is opened, the EA recalculates a single Take Profit (TP) for all orders on that side, ensuring they all close for a net profit.

### B. Hedging Mode (Crisis Management)
*   **Trigger:** Total Buy or Sell lots reach `MaxLots`.
*   **Goal:** Capital preservation and lot reduction.
*   **Flow:**
    1.  **Full Lock:** Immediately opens a trade to balance Buy/Sell lots to a 1:1 ratio.
    2.  **Order Cleanup:** Deletes all pending orders and removes all Take Profits.
    3.  **Hedge Reduction:** Waits for price movement (`PipsToTransfair`) to close profitable chunks of the hedge and use that profit to slowly delete losing trades.
    4.  **Exit:** Returns to Normal Mode only when total lots drop below `MinLots`.

---

## 2. Dynamic Profit Management (Transfair Logic)
Version 1.11 uses a unique "Selective Profit Harvesting" system called **Transfair**:
-   **Condition:** If price moves favorably by `PipsToTransfair` pips.
-   **Action:** 
    -   Close all trades that are currently in profit.
    -   Use a percentage (`PF_Percent`) of that closed profit to simultaneously close the oldest/worst losing trades.
-   **Result:** This "shaves off" the largest losses and reduces the overall lot size of the grid, preventing the EA from hitting `MaxLots` too quickly.

---

## 3. Global Logic Flow (OnTick)
On every tick, the EA performs these checks in order:

1.  **Manual Controls:** Checks if the "Close All" or "Transfair" buttons were pressed on the chart.
2.  **Asset Info:** Updates the dashboard with current Profit, Lots, and Cycle time.
3.  **Management Layer:** 
    -   If in **Hedge Mode**: Executes `ManageMainCycleReduction` and ensures the lock remains 1:1.
    -   If in **Normal Mode**: Resets pending orders, syncs TP, and checks `TARGET` profit levels.
4.  **Grid Layer:** Checks if the next pending order in the chain needs to be placed.

---

## 4. Key Input Parameters Reference
-   **Step:** Distance between grid lines.
-   **TakeProfit:** Target profit for each grid cycle in pips.
-   **Multiplier:** The scale of lot increase for recovery.
-   **MaxLots:** The safety threshold that triggers Hedging Mode.
-   **TARGET:** Total dollar profit target to close the entire cycle.
-   **PF_Percent:** How much profit to use for deleting losing trades during reduction.
